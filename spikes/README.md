# FlagDeck R0 spikes

本目录只包含 `PROJECT_PLAN.md` v3.1 要求的五个运行时可行性验证。R0 不建立产品页面、产品数据库或正式 Adapter。

## 统一原则

- 网络 fixture 只绑定动态 `127.0.0.1` 端口。
- 运行数据目录使用 `0700`，敏感文件使用 `0600`。
- 子进程由绝对路径与 argv 数组启动。
- 原始证据保存为结构化文件；秘密仅保存脱敏扫描结论。
- 每个 spike 通过代码、自动测试、实机证据和 ADR 四层给出 `PASS`、`FAIL` 或 `BLOCKED`。
