# Security Policy

## 报告安全问题

请使用 GitHub Private Vulnerability Reporting 提交安全问题。报告中应包含受影响版本、复现条件、影响范围和可行的缓解方式。

公开 Issue 不适合提交漏洞细节、真实目标、凭据、Token、Cookie、完整日志或未脱敏的截图。

## 支持范围

安全修复以最新源码和最新 GitHub Release 为准。第三方工具的漏洞由对应上游项目处理；FlagDeck 会根据影响更新 Tool Pack、固定版本或禁用相关入口。

## 使用范围

FlagDeck 仅用于 CTF、实验环境和已经取得明确授权的安全测试。使用者负责确认测试权限、目标范围和当地法律要求。
